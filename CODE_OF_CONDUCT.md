# Code of conduct

This project adopts the **Contributor Covenant, version 2.1**, as its
code of conduct.

The canonical text lives at:

  https://www.contributor-covenant.org/version/2/1/code_of_conduct/

It is reproduced verbatim in many OSS projects; rather than fork the
text into this repo (and risk drift if the upstream is updated), this
file references the canonical source. The version pinned here is
2.1, dated 2021-12-01.

## Scope

The code of conduct applies to all project spaces, including:

- This GitHub repository (issues, pull requests, discussions, code review)
- Project documentation and external write-ups about ContentOps
- Any project-affiliated communication channel (mailing list, chat,
  conference talk, sponsored event)
- Public spaces where someone represents the project or its community

## Reporting

To report a violation of the code of conduct, contact a project
maintainer privately. The current maintainers are listed in
[`MAINTAINERS.md`](MAINTAINERS.md); reach out via a GitHub direct
message or a private GitHub channel.

Reports are confidential. We will respond within 5 business days,
investigate, and take any action we judge proportionate up to and
including removing the contributor from project spaces.

Security vulnerabilities go through a different channel — see
`SECURITY.md`.

## Enforcement guidelines

Enforcement follows the four-tier guidelines from the Contributor
Covenant 2.1 §4 (Correction / Warning / Temporary Ban / Permanent
Ban). The maintainers will pick the tier proportional to the
infraction and prior history.

## Maintainer accountability

Maintainers who fail to follow or enforce this code of conduct in
good faith may face temporary or permanent repercussions as
determined by other members of the project's leadership. See
`MAINTAINERS.md` for the current maintainer set.
